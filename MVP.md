# Agentic CMS: MVP статического ядра (SaaS для локального бизнеса)

## 1. Концепция проекта
Мы создаем универсальный шаблон статического сайта (Boilerplate) для платформы Agentic CMS. Целевая аудитория — малый локальный бизнес (кафе, рестораны, студии йоги).
Управление контентом будет осуществляться не через традиционную админ-панель (GUI), а облачным ИИ-агентом на базе **Gemini CLI**, который будет удаленно редактировать Markdown-файлы через GitHub MCP Server, управляясь через естественный язык (LUI).
Текущая задача: создать масштабируемый фундамент клиентского сайта.

## 2. Архитектура данных (Контентные сущности)
Сайт должен быть построен на строгой системе коллекций контента (Content Collections), вдохновленной архитектурой Drupal. База данных состоит из Markdown и JSON файлов, разделенных на следующие сущности:

* **pages:** Уникальные страницы со сложной версткой (Главная, Контакты, О нас).
* **nodes:** Типовой контент (позиции меню кафе, занятия йогой, новости).
* **blocks:** Переиспользуемые фрагменты (промо-баннеры, врезки с отзывами).
* **taxonomy:** Словари и термины для группировки (категории меню, уровни сложности занятий).
* **staff:** Профили сотрудников (шеф-повара, тренеры).
* **settings:** Глобальные конфигурации в формате JSON/YAML (главное навигационное меню, часы работы, контактные данные).

## 3. Инструкция для ИИ-архитектора (Antigravity)
Ты — ведущий AI-разработчик. Твоя задача — спроектировать код этого шаблона, не принимая самостоятельных решений о стандартах безопасности и правилах платформы. Навыки, которые ты напишешь, предназначены для исполнения в **Gemini CLI**, а не тобой.

**Твой пошаговый план:**
1. Обязательно прочитай системные ограничения и стек технологий в файле `AGENTS.md`.
2. Используй инструмент MCP `notebooklm`. Обратись к блокноту **"The Future of Enterprise UI: From GUI to Agentic AI"** (ID: 3a262790-c619-4a44-8f5e-99e0b82f0d3b), чтобы получить архитектурные требования и структуру для Agentic CMS. Также загляни в **"Modern Web Rendering: Astro, Hugo, and Next.js Guide"** (ID: 0cceefd6-1d94-4143-b0fa-d402a9b93cff) за лучшими практиками проектирования.
3. Разверни базовый проект Astro.js с поддержкой Tailwind CSS.
4. Настрой типизацию `src/content/config.ts` для всех шести сущностей, описанных в разделе "Архитектура данных".
5. Настрой базовый роутинг, который сможет динамически генерировать страницы из папок `pages` и `nodes`.
6. Создай файл GitHub Actions (`.github/workflows/deploy.yml`) для статической сборки и деплоя на GitHub Pages.
7. Остановись и выведи отчет о проделанной работе мне на проверку.

## 4. Возможные улучшения (Roadmap)
По просьбе пользователя запланированы следующие архитектурные улучшения:
* **Модуль мультивалютности:** Создание отдельного раздела или интеграции API для актуальных курсов обмена (Exchange Rates) с возможностью автоматического пересчета цен в зависимости от выбранного языка/региона, управляемого политикой компании.
* **Подключение Web-шрифтов:** Внедрение Google Fonts (например, Inter или Roboto) вместо локальных системных шрифтов (`font-sans`), чтобы гарантировать корректное кросс-платформенное отображение спецсимволов, таких как знак рубля (₽). До решения этой задачи на некоторых Linux-системах знак рубля может отображаться некорректно (как базовый знак валюты `¤`).
